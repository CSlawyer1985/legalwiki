## 📖 项目简介

法律概念 Wiki 是一个纯静态的法律知识查询网站。所有文章由 AI 从 43,000+ 篇原始法律资料中自动提取、整理、交叉引用，覆盖合同法、公司法、刑法、建设工程、知识产权等 22 个法律领域。

灵感来源于 [Karpathy LLM Wiki](https://github.com/karpathy/llm.wiki) 方法论 —— 用 LLM 将海量原始资料提炼为结构化知识库，并通过 wikilink 交叉引用构建知识图谱。

### 核心价值

- 🔍 **全文搜索** — 客户端模糊搜索，毫秒级响应，无需服务端
- 🔗 **交叉引用** — 概念间 wikilink 互联，自动计算反向链接
- 📂 **领域导航** — 22 个法律领域分类，侧边栏折叠导航
- 📊 **知识图谱** — 高引用概念、桥接概念（跨领域）自动发现
- 🚀 **纯静态** — 零依赖部署，GitHub Pages / Vercel / 任意静态托管

## ✨ 核心特性

### 🔍 全文搜索

- 客户端模糊匹配，无需后端服务
- 搜索索引预生成（`search-index.json`），加载即搜
- 支持概念名、领域、正文内容匹配

### 🔗 Wikilink 交叉引用

- 文章中 `[[概念名]]` 自动转换为可点击链接
- 自动计算反向链接（谁引用了这个概念）
- 关联概念标签展示

### 📂 领域导航

- 侧边栏按领域折叠分组，快速定位
- 每个领域独立索引页（`domain/{领域}.html`）
- 全部领域浏览页（`browse.html`）

### 📊 首页仪表盘

- 高引用概念 TOP 10
- 桥接概念（跨领域高频引用）
- 最近更新列表
- 全领域概览卡片

### 📱 响应式设计

- 桌面端：侧边栏 + 主内容区双栏布局
- 移动端：侧边栏可折叠，自适应单栏

### ⚡ 纯静态零依赖

- 无框架、无构建工具、无运行时依赖
- 部署到任意静态托管即可运行
- 访客计数器使用不蒜子（可选）

## 🏗️ 核心架构

```
┌─────────────────────────────────────────────────┐
│          43,000+ 篇原始法律资料（只读源）          │
│          /Users/CS/Trae/知识库/wiki/concepts/     │
└──────────────────────┬──────────────────────────┘
                       │ build.py（5 步构建）
                       ▼
┌─────────────────────────────────────────────────┐
│  [1/5] 复制源文件到临时缓存                        │
│  [2/5] 解析 Markdown → 提取 frontmatter/wikilink │
│  [3/5] 生成 1,030 篇概念文章 HTML                 │
│  [4/5] 生成 22 个领域索引页 + 浏览汇总页           │
│  [5/5] 生成首页 + 搜索索引 + 清理缓存              │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│              静态站点输出                          │
│  index.html          — 首页仪表盘                │
│  browse.html         — 全部领域浏览              │
│  article/*.html      — 1,030 篇概念文章           │
│  domain/*.html       — 22 个领域索引页            │
│  search-index.json   — 全文搜索索引 (6.8MB)       │
│  style.css           — 全站样式                  │
│  app.js              — 搜索 + 交互逻辑           │
└─────────────────────────────────────────────────┘
```

### 构建流程

1. **数据拷贝** — 从只读源目录复制到临时缓存（不修改原始资料）
2. **解析** — 解析 YAML frontmatter、提取 wikilink、Markdown → HTML
3. **文章生成** — 生成带目录、反向链接、关联概念的文章页
4. **领域页生成** — 按领域分组，生成索引页和浏览汇总页
5. **首页生成** — 生成仪表盘、搜索索引，清理临时缓存

## 🚀 快速上手

### 在线访问

直接打开 `index.html` 即可在浏览器中使用，或部署到任意静态托管平台。

### 本地构建

```bash
# 1. 克隆仓库
git clone https://github.com/CSlawyer1985/legalwiki.git
cd legalwiki

# 2. 创建虚拟环境并安装依赖
python3 -m venv .venv
source .venv/bin/activate
pip install markdown

# 3. 构建站点
python build.py

# 4. 打开预览
open index.html
```

### 部署

项目为纯静态文件，可部署到任意静态托管平台：

- **GitHub Pages** — 推送到 `gh-pages` 分支即可
- **Vercel** — 连接仓库，自动部署
- **Netlify** — 拖拽文件夹即可
- **任意服务器** — 放到 Nginx/Apache 静态目录
